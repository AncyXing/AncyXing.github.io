1.准备工作

准备三个节点，要求配置好主机名称，服务器系统时间保持一致

在/etc/hostname和/etc/hosts配置主机名称

2.上传zk到三台服务器节点

解压到/usr/local下

进行解压： tar -zxvf zookeeper.tar.gz
重命名：mv zookeeper.tar.gz zookeeper
修改环境变量：vim /etc/profile
添加zookeeper的全局变量：
export ZOOKEEPER_HOME=/usr/local/zookeeper
export PATH=.:$ZOOKEEPER_HOME/bin

export PATH=.:$JAVA_HOME/bin:$ZOOKEEPER_HOME/bin

刷新环境变量：source /etc/profile
修改zookeeper的配置文件
cd /usr/local/zookeeper/conf
mv zoo_sample.cfg zoo.cfg
修改zoo.cfg配置文件
修改数据的dir:dataDir=/usr/local/zookeeper/data
修改集群地址:
server.0=yx1:2888:3888
server.1=yx2:2888:3888
server.2=yx3:2888:3888
增加服务器标识配置，需要两个步骤，第一是创建文件夹和文件，第二是添加配置内容
创建文件夹：mkdir /usr/local/zookeeper/data
创建文件myid 路径应该创建在/usr/local/zookeeper/data下面：vim /usr/local/zookeeper/data/myid
注意这里每一台服务器的myid文件的内容不同，分别修改里面的值为0，1，2；与之前的zoo.cfg配置文件里的server.0，server.1，server.2顺序对应，然后保存退出
到此为止，Zookeeper集群环境搭建完成 启动Zookeeper命令
启动路径：/usr/local/zookeeper/bin 也可以在任意目录，因为配置了环境变量
执行启动命令：zkServer.sh start
查看状态：zkServer.sh status 检验zk的mode，会看到一个leader和两个follower
ZKCli.sh进入zookeeper客户端(zookeeper数据是树型存储的)
查找：ls / ,ls /zookeeper
创建和赋值：create /imooc zookeeper
获取：get /imooc
设值：set /imooc zookeeper1314
ps：任意节点都可以看到zookeeper集群的数据一致性
ps；创建节点有两种类型，瞬时节点、持久节点

开机启动
cd /etc/rc.d/init.d/
touch zookeeper
chmod 777 zookeeper

开机启动配置
chkconfig zookeeper on/ chkconfig zookeeper off 把zookeeper添加到开机启动里面/或者移除
验证：chkconfig –add zookeeper 把zookeeper添加到开机启动里面
chkconfig –list zookeeper
这个时候就可以用service zookeeper start/stop 来启动停止zookeeper服务了
